# CredixSA Cache API

HTTP service for reading the CredixSA SQLite cache generated by Kestra warmup flows.

## Runtime

- `CREDIXSA_CACHE_DB_PATH`: SQLite path. Default: `/data/credixsa-cache/credixsa.sqlite`
- `CREDIXSA_CACHE_MAX_AGE_DAYS`: freshness window. Default: `7`
- `CREDIXSA_CACHE_API_TOKEN`: optional bearer or `X-API-Token` token.

## Endpoints

- `GET /health`
- `POST /credixsa/cache`

Request:

```json
{"cuit":"20-12345678-3","nombre":"Juan Perez"}
```

On cache hit, the response mirrors the Kestra CredixSA output shape consumed by `web/herramientas`.
On miss, it returns `status=cache_miss` and HTTP `404`.
